## Prioritization Frameworks

### ICE Scoring

Score each task on three factors (1-10):

| Factor | Description | Question to Ask |
|--------|-------------|-----------------|
| **Impact** | Value if completed | "What changes if this gets done?" |
| **Confidence** | Certainty of impact | "How sure am I about this?" |
| **Ease** | Effort required | "How much effort does this need?" |

**ICE Score = (Impact + Confidence + Ease) / 3**

Prioritize highest scores first.

---

### MoSCoW Method

| Category | Definition | % of Effort |
|----------|------------|-------------|
| **Must Have** | Non-negotiable, fails without | 60% |
| **Should Have** | Important but not critical | 20% |
| **Could Have** | Nice to have, if time permits | 15% |
| **Won't Have** | Explicitly out of scope | 5% |

---

### Ivy Lee Method (Daily)

End of each day:
1. Write 6 most important tasks for tomorrow
2. Prioritize them 1-6
3. Tomorrow: Work on #1 until complete
4. Move to #2, and so on
5. Move unfinished tasks to next day
6. Repeat daily

---

### Warren Buffett's 5/25 Rule

1. Write down 25 goals/tasks
2. Circle the top 5 most important
3. The other 20 = "avoid at all cost" list
4. These 20 are dangerous distractions

## Decision Helpers

### When Everything Feels Urgent

Ask:
- "If I could only do ONE thing today, what would it be?"
- "What will have the biggest impact in a week?"
- "What's the cost of delaying each task by one day?"

### When Stuck Between Two Tasks

| Criterion | Task A | Task B |
|-----------|--------|--------|
| Hard deadline? | | |
| Impact level? | | |
| Blocks others? | | |
| Energy required? | | |
| Current energy? | | |

Choose the one that wins more criteria.

### Quick Heuristics

- **2-Minute Rule**: <2 min? Do it now
- **3-Task Rule**: Max 3 must-do items/day
- **Tomorrow Test**: "Will this matter tomorrow?"
- **Default No**: Say no unless compelling reason

## Common Mistakes

### Mistake 1: Treating all tasks equally
**Fix**: Force rank - no ties allowed

### Mistake 2: Over-scheduling
**Fix**: Plan 60% of day, leave 40% buffer

### Mistake 3: Starting with easy tasks
**Fix**: Eat the frog - hardest task first

### Mistake 4: Ignoring energy levels
**Fix**: Match tasks to energy

### Mistake 5: Never saying no
**Fix**: Every yes = no to something else

### Mistake 6: Confusing busy with productive
**Fix**: Ask "What did I accomplish?" not "Was I busy?"

## Delegation Framework

### Delegate If:
- Someone else can do it
- It's not using your unique skills
- It's repeatable and teachable
- It helps someone else grow

### Delegation Template

```
TASK: [What needs to be done]
OWNER: [Who's responsible]
DEADLINE: [When it's due]
SUCCESS: [What "done" looks like]
RESOURCES: [What they need]
CHECK-IN: [When you'll follow up]
```
