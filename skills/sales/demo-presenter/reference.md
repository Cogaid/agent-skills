## Demo Psychology

### Why Demos Fail

1. **Information overload** - Too many features, not enough focus
2. **No connection to value** - Features without benefits
3. **Wrong audience approach** - Same demo for everyone
4. **No engagement** - Monologue vs. dialogue
5. **Weak close** - No clear next step

### The "Aha Moment"

Every great demo has 1-3 "aha moments" where the prospect says:
- "That's exactly what we need"
- "I didn't know that was possible"
- "That would save us so much time"

**Building Aha Moments:**
1. Identify their biggest pain point
2. Save that feature for middle of demo
3. Build anticipation
4. Reveal with impact statement
5. Pause to let it sink in

### Engagement Checkpoints

**Every 5 minutes, ask:**
- "Does this make sense?"
- "Is this relevant to what you're working on?"
- "How would your team use this?"
- "Any questions so far?"

## Audience-Specific Strategies

### Executive Audience

**What they care about:**
- ROI and business impact
- Competitive advantage
- Risk mitigation
- Speed to value
- Strategic alignment

**Demo approach:**
- Lead with outcomes
- Show dashboards/reports first
- Keep it high-level (15-20 min)
- Use business language
- Have data ready

**Sample opening:**
```
"[Name], I know your time is valuable, so let me focus on
what matters most: how [solution] can help [Company]
achieve [their strategic goal] faster.

At a high level, here's what companies like yours typically see:
[Metric 1], [Metric 2], [Metric 3]."
```

### Manager Audience

**What they care about:**
- Day-to-day usability
- Team adoption
- Implementation effort
- Support and training
- Integration with current tools

**Demo approach:**
- Focus on workflow improvements
- Show ease of use
- Address training concerns
- Demo integrations
- Include use case scenarios

**Sample opening:**
```
"I know you're on the front lines with your team every day,
so I want to show you exactly how [solution] would fit into
your current workflow and make your team more effective."
```

### Technical Audience

**What they care about:**
- Architecture and security
- APIs and integrations
- Customization options
- Data handling
- Performance and scalability

**Demo approach:**
- Be prepared for deep dives
- Show API documentation
- Discuss security certifications
- Address data privacy
- Allow time for technical Q&A

**Sample opening:**
```
"I know you'll want to understand how this works under the hood,
so I've prepared to go as deep as you'd like on architecture,
security, and integrations."
```

### Committee/Group Audience

**What they care about:**
- Varies by person
- Consensus building
- Risk reduction
- Addressing all concerns

**Demo approach:**
- Acknowledge all stakeholders
- Ask each person's priorities
- Balance depth vs. breadth
- Watch body language
- Address skeptics directly

**Sample opening:**
```
"I'm excited to have all the key decision-makers here.
To make sure this is valuable for everyone, let me quickly
understand what each of you is most interested in seeing today."
```

## Technical Demo Best Practices

### Environment Preparation

**Demo account setup:**
- Use realistic data
- Pre-populate with relevant examples
- Clean up test/messy data
- Prepare specific scenarios
- Have reset plan if needed

**Browser setup:**
- Clear cache and cookies
- Disable notifications
- Close other tabs
- Zoom to 100-110%
- Hide bookmarks bar

### Screen Sharing Tips

**Best practices:**
- Share specific window, not desktop
- Check for sensitive info
- Turn off message notifications
- Use spotlight/zoom feature
- Move cursor slowly and deliberately

### When Things Go Wrong

**Page won't load:**
```
"Technology! Let me try refreshing...
While that loads, let me tell you about [relevant feature]..."
```

**Feature not working:**
```
"Interesting - that's not behaving as expected. Let me show you
this another way... [alternative path]"
```

**Need to recover:**
```
"Actually, let me take a step back and show you the typical workflow,
then we'll come back to this specific scenario."
```

## Objection Handling During Demo

### Common Objections and Responses

**"We already have something similar"**
```
"That makes sense - most companies have some solution in place.
What specifically is working well, and where are the gaps?

[After response]

That's actually where we see the biggest difference.
Let me show you how we handle [their gap]..."
```

**"This looks complicated"**
```
"I appreciate that feedback. What specifically feels complicated?

[After response]

Actually, most users get comfortable within [timeframe].
Here's why - [show simple workflow]..."
```

**"How long does implementation take?"**
```
"Great question. Typical implementation is [timeframe].
Here's the process:

1. [Phase 1] - [time]
2. [Phase 2] - [time]
3. [Phase 3] - [time]

We have dedicated support throughout."
```

**"What about security?"**
```
"Security is critical for us too. We're [certifications].
Specifically:

• Data encryption: [details]
• Access controls: [details]
• Compliance: [relevant standards]

I can send you our security whitepaper after the call."
```

**"We need to think about it"**
```
"Of course - this is an important decision.
What specific aspects do you need to think through?

[After response]

Would it help if I [specific action] to help with
your evaluation?"
```

### The LAER Framework for Objections

**L - Listen**
- Let them fully express concern
- Don't interrupt
- Take notes

**A - Acknowledge**
- "I understand..."
- "That's a valid concern..."
- "Many customers have asked that..."

**E - Explore**
- "Tell me more about..."
- "What's driving that concern?"
- "Has that been an issue before?"

**R - Respond**
- Address specifically
- Provide evidence
- Check if resolved

## Follow-Up Framework

### Same Day Follow-Up

**Within 2 hours of demo:**

```
Subject: Great connecting, [Name] - next steps inside

Hi [Name],

Thanks for the time today exploring [solution] for [Company].

Key Takeaways:
• [Priority 1 they mentioned] - We discussed [relevant capability]
• [Priority 2 they mentioned] - Showed [relevant feature]
• [Priority 3 they mentioned] - Covered [relevant aspect]

As Promised:
• [Resource 1 mentioned]
• [Resource 2 mentioned]

Next Step:
[Agreed action] by [date]

Let me know if any questions come up in the meantime.

[Signature]
```

### 24-Hour Follow-Up

**If no immediate next step was set:**

```
Subject: Following up on [topic] demo

Hi [Name],

Wanted to check in after yesterday's demo.

Based on our conversation, it seemed like [their priority]
was top of mind for your team.

I put together [resource] that digs deeper into how
[similar company] tackled that challenge.

When would work for a quick follow-up to discuss
your evaluation process?

[Signature]
```

### Post-Demo Nurture

**Day 3:** Share relevant case study
**Day 7:** Check in on questions
**Day 14:** Offer additional demo/stakeholder meeting
**Day 21:** New angle/additional value prop

## Demo Metrics and Improvement

### Track These Metrics

| Metric | How to Track |
|--------|--------------|
| Show rate | Scheduled vs. attended |
| Demo-to-opp rate | Demos vs. qualified opps |
| Demo-to-close rate | Demos vs. closed won |
| Average demo length | Calendar/recording data |
| Questions asked | Self-reporting |

### Self-Review Questions

After each demo:
1. Did I customize to their specific needs?
2. How many questions did I ask?
3. Did I create an "aha moment"?
4. Did I end with a clear next step?
5. What would I do differently?

### Common Patterns to Fix

**If show rate is low:**
- Confirm meetings 24h and 1h before
- Send calendar invite with video link
- Include agenda and what to prepare

**If demo-to-opp is low:**
- Better qualification before demo
- More discovery during demo
- Clearer value connection

**If demo-to-close is low:**
- May have wrong stakeholders
- Not addressing real objections
- Need better follow-up process

## Remote Demo Tips

### Video Best Practices

- Camera at eye level
- Good lighting from front
- Clean background
- Professional appearance
- Look at camera when speaking

### Engagement Techniques

- Use names frequently
- Ask participants to unmute
- Use chat for questions
- Share annotated screens
- Leave video on

### Platform-Specific Tips

**Zoom:**
- Enable spotlight/pin speaker
- Use annotations
- Mute all except presenter
- Enable waiting room

**Teams:**
- Test audio settings
- Use presenter mode
- Share specific window
- Enable live captions

**Google Meet:**
- Check browser permissions
- Pin relevant participants
- Use present mode
- Enable captions
