## Purpose

Deliver engaging, outcome-focused product demonstrations that connect features to customer value and drive buying decisions.

## Quick Reference

### Demo Structure (The CLOSER Framework)

| Phase | Duration | Purpose |
|-------|----------|---------|
| **C**onnect | 5 min | Build rapport, confirm goals |
| **L**isten | 5-10 min | Understand specific needs |
| **O**utline | 2 min | Set agenda and expectations |
| **S**how | 15-20 min | Demonstrate value |
| **E**ngage | 5-10 min | Handle questions, objections |
| **R**esolve | 5 min | Define next steps |

### Demo Time Guidelines

| Audience | Ideal Length | Max Length |
|----------|--------------|------------|
| Executive | 20 min | 30 min |
| Manager | 30 min | 45 min |
| Technical | 45 min | 60 min |
| Committee | 45 min | 60 min |

## Demo Flow

### Opening (Connect & Listen)

**Build rapport:**
```
"Thanks for taking the time today! Before we dive in,
I'd love to learn a bit more about what prompted this conversation
and what you're hoping to accomplish."
```

**Confirm attendees and roles:**
```
"Just want to make sure I have everyone - looks like we have
[names]. [Name], can you help me understand your role in
evaluating solutions like ours?"
```

**Discovery questions:**
- What's driving the timeline?
- What happens if nothing changes?
- What does success look like?
- Who else is involved in this decision?

### Setting the Agenda (Outline)

```
"Based on what you've shared, here's what I'd like to cover:

1. [Relevant capability matching their priority #1]
2. [Relevant capability matching their priority #2]
3. Questions and next steps

Does that sound right? Anything else you'd like me to cover?"
```

### The Demo (Show)

**Key principles:**
1. Show, don't tell
2. Feature → Benefit → Impact
3. Use their language
4. Ask questions throughout
5. Build to "wow moments"

**Feature-Benefit-Impact Pattern:**
```
"This is [feature name]..."
"What this means for your team is [benefit]..."
"Companies like yours typically see [impact/metric]..."
```

### Handling Questions (Engage)

**Acknowledge and answer:**
```
"Great question! [Answer]. Does that address your concern?"
```

**Defer appropriately:**
```
"That's a detailed technical question - let me get you
the exact answer after this call. What I can tell you now is..."
```

### Closing (Resolve)

```
"Based on what we've discussed, it sounds like [solution]
could help you [achieve their goal].

What would be a logical next step from here?"
```

## Common Mistakes to Avoid

| Mistake | Instead |
|---------|---------|
| Feature dumping | Show 3-5 key features that match their needs |
| Talking too much | Ask questions, get them involved |
| Ignoring time | Check in at halfway point |
| Generic demo | Customize to their use case |
| No clear next step | Always end with specific action |
