## Chat Etiquette

### Tone Guidelines

**DO:**
- Use customer's name (once at start, once at end)
- Write in complete sentences
- Use positive language
- Be concise but not curt
- Match customer's communication style

**DON'T:**
- Use ALL CAPS
- Over-use exclamation points!!!
- Use jargon or acronyms
- Sound robotic or scripted
- Leave long pauses without explanation

### Message Structure

**Ideal message length:** 1-3 sentences per message

**For complex explanations:**
```
Let me break this down:

1. First, [step one]
2. Then, [step two]
3. Finally, [step three]

Does this make sense so far?
```

### Typing Indicators

- Start typing immediately to show presence
- If researching, send: "Let me look into this for you - one moment please"
- Update every 60-90 seconds during research

## Handling Difficult Situations

### Angry Customer

**Pattern: HEAT**
- **H**ear them out (let them vent)
- **E**mpathize ("I completely understand your frustration")
- **A**pologize sincerely
- **T**ake action immediately

**Example:**
```
I can hear how frustrated you are, and honestly, I would be too.
I'm really sorry this happened. Let me fix this right now.
[Take immediate action]
```

### Confused Customer

**Pattern: CLEAR**
- **C**larify what they're trying to do
- **L**isten for the real issue
- **E**xplain in simple terms
- **A**sk if they understood
- **R**epeat if necessary

**Example:**
```
Just to make sure I understand correctly - you're trying to [action]
but [problem] is happening. Is that right?

[After confirmation]

Got it! Here's what we'll do...
```

### Demanding Customer

**Keep boundaries while being helpful:**
```
I really want to help you with this. While I can't [unreasonable request],
here's what I can do: [alternative solution]. Would that work for you?
```

### Customer Threatening to Leave

**Retention approach:**
```
I'd hate to see you go - you've been with us for [duration].
Before you decide, can you share what's driving this?
I may be able to help resolve the issue or find a solution that works better for you.
```

## Chat Scenarios

### Billing Issue

```
Customer: I was charged twice for my subscription

Agent: Oh no, I'm sorry to see that double charge! Let me pull up your
account right now and get this sorted.

[Research]

I can see the duplicate charge from [date]. I'm processing a refund
for $[amount] right now. You should see it back in your account within
3-5 business days.

Is there anything else I can help you with while we're connected?
```

### Technical Issue

```
Customer: The app keeps crashing

Agent: I'm sorry you're experiencing crashes - that's definitely frustrating!
Let's get this fixed.

A few quick questions:
1. What device/browser are you using?
2. When did this start happening?
3. Does it crash on a specific action or randomly?

[After gathering info]

Based on what you've described, let's try [solution].
Can you try that while we're chatting so I can help if needed?
```

### Feature Request

```
Customer: You should really add [feature]

Agent: That's actually a great suggestion! I don't have a timeline for that
feature, but I'd love to log this feedback for our product team.

Can you tell me a bit more about how you'd use this? It helps me make
a stronger case for your request.

[Log feedback after gathering details]

I've submitted this to our product team with your use case.
While I can't promise timelines, your feedback genuinely helps shape our roadmap.
```

## Multitasking Best Practices

### Handling Multiple Chats

**Setup:**
- Arrange chat windows for easy visibility
- Use keyboard shortcuts for switching
- Keep canned responses easily accessible

**Prioritization:**
1. Customer who just responded
2. Customer waiting longest
3. New incoming chat

**Time Management:**
```
If you need time:
"Great question! Let me check on that for you - I'll have an answer
in about 2 minutes. Feel free to grab a coffee ☕"
```

### When to Single-Task

Focus on one chat when:
- Customer is extremely upset
- Issue is complex and requires deep focus
- You're handling a VIP or high-value customer
- Conversation involves sensitive information

## Chat Metrics

### Key Performance Indicators

| Metric | Target | How to Improve |
|--------|--------|----------------|
| First Response Time | < 30 sec | Use greeting templates, monitor queue |
| Average Handle Time | < 10 min | Use canned responses, know when to escalate |
| Resolution Rate | > 80% | Improve product knowledge, clear escalation |
| Customer Satisfaction | > 90% | Empathy, speed, clear communication |
| Chats per Hour | 4-6 | Efficient templates, good multitasking |

### Quality Indicators

**Good chat characteristics:**
- Customer issue clearly understood
- Solution provided or clear next steps
- Positive closing
- No unresolved questions

## Canned Response Library

### Greetings

```
[greeting-standard]
Hi [Name]! Thanks for reaching out. I'm [Agent] and I'll be helping you today. How can I assist you?

[greeting-queue]
Hi [Name], thanks for waiting! I'm [Agent] and I'm here to help. What can I do for you?

[greeting-returning]
Welcome back, [Name]! Great to see you again. How can I help you today?
```

### Hold/Delay Messages

```
[hold-short]
Give me just a moment to look into this for you.

[hold-medium]
I'm researching this now - should have an answer in about 2 minutes. Thanks for your patience!

[hold-long]
This is taking a bit longer than expected. I want to make sure I give you accurate information. Just another minute or two!
```

### Transitions

```
[transfer-intro]
I'm going to connect you with our [department] team who can better assist with this. One moment while I bring them in.

[escalate]
This needs some additional attention. I'm escalating this to [team/person] who will reach out within [timeframe].
```

### Closings

```
[close-resolved]
Glad I could help! Is there anything else I can assist you with today?

[close-final]
Thanks for chatting with us today, [Name]! Have a great [day/evening]!

[close-followup]
I've [taken action]. You'll receive [confirmation/follow-up] within [timeframe]. Anything else I can help with?
```

### Common Issues

```
[password-reset]
No problem! I've sent a password reset link to [email]. It should arrive within a few minutes. If you don't see it, check your spam folder.

[billing-question]
Let me pull up your billing information. [Pause] I can see [details]. Would you like me to explain any of these charges?

[feature-how]
Great question! Here's how to [action]:
1. [Step 1]
2. [Step 2]
3. [Step 3]
Let me know if you run into any issues!
```

## Escalation Triggers

Escalate immediately when:
- Customer threatens legal action
- Data breach or security concern
- Customer mentions self-harm
- VIP/enterprise customer with complex issue
- Issue requires system access you don't have
- Customer explicitly requests supervisor

**Escalation message:**
```
I want to make sure you get the best possible help with this.
I'm connecting you with [person/team] who has [expertise/access]
to fully resolve this for you. Please hold for just a moment.
```
